# 📊 TutorNow Analytics Project

## 🧠 Overview
This project simulates a real-world tutoring platform called **TutorNow**, based in Sydney, Australia. The goal is to design and implement a complete data pipeline to analyze student demand, tutor supply, pricing trends, and engagement behavior.

The project demonstrates an end-to-end workflow from raw data generation and cleaning to data warehousing and business intelligence visualization.

---

## 🎯 Objectives
- Analyze student distribution across Sydney suburbs  
- Evaluate tutor supply vs demand by subject and location  
- Identify pricing trends based on experience and subject  
- Understand booking patterns and seasonal trends  
- Measure student retention and tutor performance  

---

## 🏗️ Architecture
CSV Dataset → Python (Cleaning & Feature Engineering) → Snowflake (Data Warehouse) → Power BI (Dashboard)


---

## 🛠️ Tech Stack

- **Python (Pandas)** → Data cleaning & feature engineering  
- **Snowflake** → Cloud data warehouse & SQL transformations  
- **Power BI** → Interactive dashboards & visualization  
- **SQL** → Data modeling (fact & dimension tables)  

---
## 📊 Dashboard (Power BI)

The Power BI dashboard includes:

### 📌 Overview
- Total sessions, revenue, completion rate  
- Demand by suburb (map visualization)

### 💰 Pricing Insights
- Hourly rate vs experience  
- Pricing distribution by subject  

### 📈 Demand Trends
- Sessions over time (monthly trends)  
- Peak booking days  

### 🔁 Retention & Performance
- Repeat student rate  
- Tutor rating vs student retention  

---

## 🔍 Key Insights

- High student demand observed in suburbs like Parramatta and Sydney CBD  
- STEM subjects (especially Coding) command higher hourly rates  
- Tutors with higher ratings tend to retain more students  
- Peak demand occurs during weekdays and exam periods  
- Discounts have moderate impact on booking volume
